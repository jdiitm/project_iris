# RFCs

This directory contains formal requirements and design documents for Project Iris.

## Active RFCs

| RFC | Title | Status |
|-----|-------|--------|
| [RFC-001](RFC-001-SYSTEM-REQUIREMENTS.md) | System Requirements | Draft |

## RFC Process

1. **Draft** — Initial proposal, open for feedback
2. **Review** — Active review by stakeholders
3. **Approved** — Accepted as authoritative
4. **Superseded** — Replaced by newer RFC

## Purpose

RFCs are the **single source of truth** for:
- System requirements and guarantees
- Architecture decisions
- API contracts
- Operational expectations

All code should reference RFCs for requirements, not audit documents.
